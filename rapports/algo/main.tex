\section{Algorithmes de génération}

\subsection{Distance entrée / sortie}

\textbf{Cet algorithme sera utilisé dans les algorithmes à venir.}

Le placement d'une distance entrée / sortie sera ici considéré comme un algorithme à part. Le respect strict de la distance nécessite en effet de réaliser un parcours intelligent vers la sortie, depuis l'entrée, la distance les séparant.

\subsubsection{Algorithme utilisé}

Une approche par niveaux sera utilisée ici. L'algorithme sera un \textbf{parcours en largeur}.

\newpage

\subsubsection{Pseudo-code}

On note :
\begin{itemize}
\item \( d \) : la distance cible entre l'entrée et la sortie.
\end{itemize}

\begin{codeboxlang}{pseudocode}
créer une position aléatoire entrée dans le labyrinthe

créer une file vide
créer une map distances vide
créer une liste positionsResultat vide
ajouter entrée à la file
ajouter (entrée, 0) à distances
niveauActuel = 0
distanceReelle = 0

tant que la file n'est pas vide :
    tailleNiveau = taille de la file
    
    si niveauActuel > d :
        arrêter le parcours
    
    si niveauActuel = d :
        pour i de 0 à tailleNiveau :
            position = retirer premier élément de la file
            ajouter position à positionsResultat
            distanceReelle = niveauActuel
            arrêter le parcours
    
    pour i de 0 à tailleNiveau :
    positionCourante = retirer premier élément de la file
    pour chaque direction {haut, bas, gauche, droite} :
        prochaine = calculer position adjacente selon direction
        si prochaine valide et prochaine non visitée et pas de mur :
            ajouter (prochaine, niveauActuel + 1) à distances
            ajouter prochaine à la file
    
    distanceReelle = niveauActuel
    niveauActuel = niveauActuel + 1

si positionsResultat est vide :
    pour chaque (position, distance) dans distances :
    si distance = distanceReelle :
    ajouter position à positionsResultat

sortie = choisir aléatoirement une position dans positionsResultat
définir entrée dans le labyrinthe
définir sortie dans le labyrinthe
mettre à jour la distance effective avec distanceReelle
\end{codeboxlang}

Notre solution permet alors un strict respect de la distance entrée / sortie, et surtout, un calcul de la distance maximale possible afin de ramener la distance voulue à celle maximale si celle voulue était impossible dans ce labyrinthe. Cette notion est \textbf{importante pour la suite}.

La mise à jour de la distance effective permet, entre autres, de prévenir la structure de labyrinthes que la distance voulue a changé, permettant un retour utilisateur.

\subsubsection{Structures de données utilisées}

Pendant l'exécution, l'algorithme utilise principalement une file et un dictionnaire.

\begin{itemize}
\item La file est indispensable puisqu'il s'agit d'un parcours en largeur. Elle nous permet d'explorer plusieurs niveaux à la fois (logique plus efficace).
\item Le dictionnaire sert à associer une position à sa distance. Elle permet 
\end{itemize}
  
Quant au retour, l'algorithme utilise un objet "distanceResultat" contenant :
  
\begin{itemize}
\item Une liste de positions.
\item La distance effective.
\end{itemize}

Puis, il choisit aléatoirement une position parmi la liste.
Sera attribuée au labyrinthe la distance effective, afin d'avertir de l'impossibilité de celle initialement voulue.

\subsubsection{Pertinence}

La motivation principale quant au choix des structures est l'efficacité. La structure choisie dans ce cas est minimale et comporte uniquement ce dont nous avons besoin pour déterminer une position aléatoire respectant la distance demandée (ou maximum si impossible).

\subsection{Labyrinthes parfaits : Version TP}

\subsubsection{Algorithme utilisé}

L’algorithme utilisé est celui du parcours en profondeur. L’objectif est de placer des murs partout dans le labyrinthe, puis de le parcourir en supprimant les murs gênants au passage. Cela garantit que le résultat sera un labyrinthe parfait.

\newpage

\subsubsection{Pseudo-code}

\begin{codeboxlang}{pseudocode}
1. Créer une position aléatoire comme entrée dans le labyrinthe
2. Créer une pile vide de positions
3. Créer un tableau de booléens "visitée" initialisé à false
4. Ajouter l'entrée à la pile
5. Marquer l'entrée comme visitée

6. Tant que la pile n'est pas vide :
    a. oldPosition = sommet de la pile (peek)
    b. directions = [haut, bas, gauche, droite]
    c. Mélanger l'ordre des directions aléatoirement
    d. positionValideTrouvée = false

    e. Pour chaque direction dans directions :
        i. position = oldPosition + direction
        ii. Si position est dans le labyrinthe ET pas encore visitée :
            - Marquer position comme visitée
            - Ajouter position à la pile
            - Supprimer le mur entre oldPosition et position
            - positionValideTrouvée = true
            - Break (sortir de la boucle des directions)

    f. Si positionValideTrouvée == false :
        - Retirer oldPosition de la pile (pop)
\end{codeboxlang}

\subsubsection{Structures de données utilisées}

Pendant l'exécution, l'algorithme utilise principalement une pile et un tableau de booléens.

\subsubsection{Pertinence}

La pile est essentielle pour le parcours en profondeur, permettant de revenir en arrière lorsque nécessaire. Le tableau de booléens permet de suivre les cellules déjà visitées, évitant ainsi les cycles et garantissant que chaque cellule est visitée une seule fois.

\subsection{Labyrinthes parfaits : Version originale - Fusion}

\subsubsection{Algorithme utilisé}

L’algorithme utilisé est celui de la fusion aléatoire. Son objectif est d’attribuer un numéro différent à chaque case. Tant que toutes les cases n’ont pas le même numéro, on choisit deux cases adjacentes et on fusionne leurs numéros. Cela génère un labyrinthe parfait, mais avec une apparence différente, généralement jugée meilleure de celle obtenue par le parcours en profondeur.

\newpage

\subsubsection{Pseudo-code}

\begin{codeboxlang}{pseudocode}
génère une grille de numéro différents

1. Générer une grille où chaque cellule a un numéro unique
2. Tant que la grille contient au moins deux numéros différents :
    a. Choisir une position aléatoire (pos)
    b. Choisir une direction aléatoire (dir)
    c. positionVoisine = pos + dir

    d. Si positionVoisine est dans le labyrinthe ET que les numéros de pos et positionVoisine sont différents :
        i. Supprimer le mur entre pos et positionVoisine
        ii. Fusionner les cellules : toutes les cases ayant le numéro de positionVoisine prennent le numéro de pos

\end{codeboxlang}

\subsubsection{Structures de données utilisées}

Pendant l'exécution, l'algorithme utilise principalement une matrice d'entiers pour les numéros des cellules.

\subsubsection{Pertinence}

La matrice d'entiers est essentielle pour suivre les numéros des cellules, permettant de déterminer rapidement si deux cellules peuvent être fusionnées. Cette structure est simple et efficace pour l'objectif de l'algorithme.

\subsection{Labyrinthes aléatoires}

L'algorithme de génération de labyrinthe aléatoire héritant de l'algorithme parfait, cette sous-section est détaillée après la précédente.

L'objectif sera de partir d'un labyrinthe parfait et de le rendre imparfait. Cette solution permet un respect des propriétés "parfaites" par héritage.

Le chemin entre l'entrée et la sortie est, par définition, garanti, puisqu'un labyrinthe parfait est un graphe connexe.

La propriété connexe demeure inchangée tant qu'aucun mur n'est ajouté.

Cet algorithme n'ajoute aucun mur, il ne fait qu'en enlever.

Puisqu'un labyrinthe aléatoire est ici un graphe connexe, il existe toujours un chemin entre l'entrée et la sortie.

\subsubsection{Algorithme utilisé}

L'algorithme consiste à, pour un pourcentage de murs noté \( p \), parcourir les murs, et, en enlever par un tirage de probabilité (1 - \( p \)).

Concernant la distance entrée / sortie, puisque sans paramètre dans le sujet, elle sera ramenée au maximum possible du labyrinthe, comme prévu par l'algorithme de placement.

\subsubsection{Pseudo-code}

On note :
\begin{itemize}
\item \( m \) : un labyrinthe parfait
\item \( p \) : le pourcentage de murs souhaité
\end{itemize}

\begin{codeboxlang}{pseudocode}
pour y allant de 0 à hauteur - 1
  pour x allant de 0 à largeur
    si mur horizontal à [y][x]
      si valeur aléatoire entre (0,1) inférieure à (1 - p)
        enlever mur horizontal à [y][x]
pour y allant de 0 à largeur - 1
  pour x allant de 0 à hauteur
    si mur vertical à [y][x]
      si valeur aléatoire entre (0,1) inférieure à (1 - p)
        enlever mur vertical à [y][x]
\end{codeboxlang}

\subsubsection{Structures de données utilisées}

Les structures utilisées sont identiques à celle de l'algorithme parfait utlisé, que ce soit celui de \textit{TP ou Fusion}.

\subsubsection{Pertinence}

La pertinence est donc identique.

\section{Complément et Bilan}

\subsection{Distance entrée / sortie}

Notre solution actuelle est optimisée, mais elle n'est apparue qu'après plusieurs tentatives. En effet, notre première solution consistait à :

\begin{itemize}
\item Faire un dictionnaire (clé : Position, valeur : Distance) de toutes les positions depuis la position en paramètre par parcours en largeur. 
\item Puis, choisir toutes les positions satisfaisant 
\end{itemize}

En réalité, cette approche est inefficace puisqu'elle implique de parcourir absolument tout le labyrinthe. Elle avait été imaginée pour provisionner une solution simple au problème du maximum de la distance entrée / sortie.

Les performances de l'algorithme parfait ont été multipliées par 10 après application de la nouvelle solution.

\subsection{Placement des entités}

Notre projet comporte des \textbf{entités} (\emph{joueurs, monstres, sorties...}) qui doivent être placées initialement selon des stratégies. Le problème étant très similaire à celui du placement de l'entrée et de la sortie, un algorithme de la sorte a été implémenté pour le calcul de leurs positions de départ.

La différence étant que le placement vérifie l'absence d'entités sur cette case. L'algorithme enlève alors les positions occupées pour le niveau exploré. Si le niveau demandé ne comporte aucune position valide, alors nous explorons un niveau supérieur.

\subsection{Déplacement des monstres}

Notre projet comporte des \textbf{monstres} qui sont des entités qui cherchera à toucher le joueur mettant ainsi fin à sa partie.

Le monstre se déplacera en fonction du meilleur chemin vers le joueur, donner par une fonction implémentant un parcours en largeur, le monstre choisira donc toujours le chemin le plus optimal vers le joueur.
